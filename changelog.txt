Bug Fixes:
  - Movie info screen: the highlighted cast member's accent frame had its top
    3px clipped. The cast strip (control 100) is a list control, which clips
    item content to its bounds, and the focused frame was drawn at top=-3 (above

    change to any visible position and no focus jitter (thumb top matches in
    both layouts). script-easymovie-info.xml only.

v1.3.0 (2026-07-05)
--------------------------
New Features:
  - Movie info screen is now EasyMovie's own fullscreen dialog instead of
    Kodi's native movie information window. The native window builds its
    content from the active skin's media-window context and rendered blank on
    Arctic-family skins (the original v1.2.0 Info-pane bug). The addon-owned
    dialog renders from the movie's full details on every skin, in EasyMovie's
    theme: poster, fanart, cast with photos, plot, and a metadata column (year,
    rating with vote count, runtime, certification, genres), plus a Play button.
    Play routes through the normal set-aware flow (INFO_RESULT_PLAY ->
    main.py play_movie), so resume and in-progress tracking keep working,
    exactly as a direct pick.
  - Skin-adaptive fonts: EasyMovie's custom dialogs now adapt their fonts to the
    active skin. Kodi resolves a dialog's <font> only against the active skin's
    font set (addons cannot register their own fonts), so on non-Estuary skins
    the Estuary-named anchors fell back to font13 and lost their size hierarchy.
    The addon now reads the active skin's Font.xml at runtime, maps its anchor
    fonts to the skin's nearest-size real text fonts, and loads font-substituted
    dialog copies, restoring the hierarchy on any skin. Estuary is unchanged
    (identity short-circuit, no generation). Works around Kodi issue #28534.

Internal:
  - Replaced the native DialogVideoInfo pane with resources/lib/ui/info_dialog.py
    (InfoDialog, a WindowXMLDialog) and resources/skins/Default/1080i/
    script-easymovie-info.xml. Added display formatters (format_runtime,
    format_rating, format_genres, format_certification, build_metadata_rows,
    build_cast_items). browse_window._show_info() fetches full details and opens
    the dialog; a Play press returns INFO_RESULT_PLAY, consumed by main.py.
  - Removed the native-pane mechanism and its tests: _build_info_listitem,
    _show_native_info, _record_native_play_if_playing, set_storage,
    RESULT_ALREADY_PLAYING, and the NATIVE_PLAY_POLL_* constants. The
    movieinformation black backdrops in the five browse-view skins are removed
    (they only masked the blank native pane).
  - New resources/lib/ui/skin_fonts.py. parse_fontset() turns a skin fontset
    into {name: size} with a DOCTYPE/ENTITY reject, a 512 KiB per-file cap, and
    a font-name allowlist applied AFTER $PARAM resolution. build_include_table()
    + _expand_include() + _resolve_params() resolve parameterized-include /
    $PARAM fontsets (e.g. Arctic Fuse), bailing safely to identity on
    $VAR/$INFO/skinvariables, missing params, conditional includes, or cycles.
    build_font_map() picks the nearest-size real font, excluding
    special-purpose (symbol/icon/glyph) faces by name. ensure_generated() writes
    font-substituted dialog XML into addon_data via an atomic PID-unique temp +
    O_EXCL lock + rename-swap, cache-keyed on skin id/version + fontset + addon
    version + max mtime across the skin's XML files, with an identity
    short-circuit for Estuary and a guaranteed fallback to the shipped path.
    Routed through every dialog scriptPath (base- and clone-safe) and tagged
    with KODI-FONT-WORKAROUND (kodi#28534) markers plus a module DELETION GUIDE
    for clean removal if Kodi ships native addon-font support.
  - Replaced the nonexistent font14_title (which silently fell back to font13)
    with the real Estuary font36_title across all dialog XMLs.
  - Test suite is now 206 tests (was 151): added tests/test_info_dialog.py and
    tests/test_skin_fonts.py.

Development (dev/CI only, not shipped in the addon):
  - Established a pinned dev toolchain and matching CI gate: ruff, pyflakes,
    pyright, vulture, and pytest+coverage run via pre-commit (commit and
    pre-push) and mirrored in CI. Added a Dependabot config (with a cooldown)
    and pinned GitHub Actions to commit SHAs.
  - Dependabot updates since v1.2.0: ruff 0.15.5 -> 0.15.18, pytest 9.0.2 ->
    9.1.1, pyright 1.1.408 -> 1.1.410, coverage 7.13.4 -> 7.14.1;
    actions/checkout 4.3.1 -> 7.0.0, actions/configure-pages 5.0.0 -> 6.0.0,
    actions/upload-pages-artifact 3.0.1 -> 5.0.0, actions/deploy-pages 4.0.5 ->
    5.0.0, actions/setup-python 5.6.0 -> 6.2.0.

v1.2.0 (2026-06-20)
--------------------------
New Features:
  - Movie Info pane in Browse Mode: pressing Info (the I key or the remote
    Info button) on a focused movie opens Kodi's native movie information
    screen, fully populated with plot, full cast with photos, director,
    writer, studio, country, ratings, and artwork. Works in all five view
    styles. Back out to keep browsing, or play directly from the pane.
  - Playing from the info pane is recorded as a started movie for the running
    instance (base addon or clone, each into its own data), so resume and
    in-progress tracking keep working, then the results window closes to match
    a normal pick. A solid black backdrop renders behind the pane so the
    EasyMovie layout does not show through.

Internal:
  - Added get_movie_full_details_query() (cast, crew, art, file path) feeding
    the native pane. browse_window._show_native_info() builds a complete
    ListItem and opens it via xbmcgui.Dialog().info();
    _record_native_play_if_playing() detects post-pane playback;
    RESULT_ALREADY_PLAYING tells main.py to exit without replaying.
  - ACTION_SHOW_INFO (11) added; Info dispatched from BrowseWindow.onAction.
    Black backdrop added to all five browse-view skins, gated on
    Window.IsVisible(movieinformation).
  - The pytest suite (151 tests) is now tracked in version control and excluded
    from the packaged addon zip. pyright excludes the _temp scratch dir.

v1.0.1 (2026-04-27)
--------------------------
Bug Fixes:
  - Fixed: Resume from the "Resume Movie?" dialog now actually starts at the
    saved position. The previous implementation called Player.Open and then
    issued xbmc.Player().seekTime() after a 500 ms sleep — but the player was
    rarely ready that quickly, so the seek was silently dropped and playback
    started from zero. Now uses Kodi's native single-item Player.Open
    options.resume with a Player.Position.Time object, which sets the file
    item's start offset before the player initialises. A 10-second rewind is
    applied for context, matching EasyTV's behavior.

Internal:
  - build_play_movie_query() in data/queries.py accepts an optional
    resume_seconds argument and emits options.resume = {hours, minutes,
    seconds} when set.
  - Added RESUME_REWIND_SECONDS constant (10 seconds).
  - Added regression tests covering the resume timecode split, including the
    exact position from the original report (3266s = 0h54m26s).
  - Verified live against Kodi 21: requested timecode is honoured, no
    post-play seek required.


v1.0.0 (2026-03-24)
--------------------------
New Features:
  - Filter Wizard: step-by-step filter flow for genre, watched status, age rating,
    runtime, time period, and score. Each filter configurable as Ask, Pre-set, or Skip.
    Supports genre ignore/select with OR/AND matching, decade browsing, recency ranges,
    and cumulative movie counts per option.
  - Browse Mode: visual results screen with 5 view styles — Showcase (horizontal
    filmstrip carousel, default), Card List, Posters, Big Screen, and Split View.
    Includes Re-roll, Surprise Me, Play Full Set, and configurable result count (1–50).
  - Playlist Mode: automatic movie marathon generation with configurable length (1–20),
    sort options, partial resume prioritization, and seek-to-resume-point.
  - Movie Set Awareness: detects Kodi movie collections and suggests the first unwatched
    entry instead of a random one. Shows set name and position in browse views.
  - Set Continuation: after finishing a collection movie, prompts to watch the next one
    with a configurable countdown timer and default action.
  - Earlier Movie Warning: alerts when about to watch a set movie while an earlier
    entry is unwatched, via background service monitoring.
  - Smart Re-suggestion: tracks recently suggested movies and avoids repeating them
    within a configurable cooldown window (4–72 hours).
  - Movie Pool Filtering: limit the movie pool to any Kodi smart playlist.
  - Clone Support: create multiple EasyMovie instances with independent settings.
    Includes mandatory version check and one-click update on launch.
  - 4 Color Themes: Golden Hour, Ultraviolet, Ember, Nightfall — applied to all
    views and dialogs. Live preview cycling via T key / blue remote button.
  - Custom Icons: set a custom addon icon per instance, persisted across upgrades.
  - In-progress Check: on launch, offers to resume a partially watched movie.
  - Wizard Memory: remembers last wizard answers for quick repeat sessions.
  - Structured Logging: separate log file with rotating backups, structured key=value
    data, and per-module loggers.
  - GitHub Pages landing page, wiki documentation, and Kodi forum thread.
  - CI validation workflow (syntax, pyflakes, pyright, kodi-addon-checker).
